Lifecycle Methods

Se necesitarás el conocimiento sobre React métodos de ciclo de vida antes de poder empezar a obtener los datos. Estos métodos son un gancho en el ciclo de vida de un componente React. pueden ser utilizados en los componentes de clase ES6, pero no en los componentes funcionales sin estado.

You will need the knowledge about React lifecycle methods before you can start to fetch data. These methods are a hook into the lifecycle of React component. They can be used in ES6 class components, but not in functional stateless components.

Vos ya conoce dos métodos de ciclo de vida de un componente de clase ES6: constructor() y render().

You already know two lifecycle methods in a ES6 class component: constructor() and render().

El constructor sólo se te llama cuando se crea una instancia del componente y se inserta en el DOM. El componente se crea una instancia. Ese proceso se llama montaje del componente.

The constructor is only called when an instance of the component is created and inserted in the DOM. The component gets instantiated. That process is called mounting of the component.

El render() método se llama durante el proceso de montaje también, pero también cuando las actualizaciones de componentes. Cada vez que el estado o los puntales de un cambio de componentes, el render() método se llama.

The render() method is called during the mount process too, but also when the component updates. Each time when the state or the props of a component change, the render() method is called.

Ahora usted sabe más sobre los dos métodos de ciclo de vida y cuando se les llama. Que ya las utilizan también. Pero hay más de ellos.

Now you know more about the two lifecycle methods and when they are called. You already used them as well. But there are more of them.

El montaje de un componente tiene dos más métodos de ciclo de vida: componentWillMount() y componentDidMount(). El constructor se llama en primer lugar, componentWillMount() se llama antes de que el render() método y componentDidMount() se llama después de que el render() método.

The mounting of a component has two more lifecycle methods: componentWillMount() and componentDidMount(). The constructor is called first, componentWillMount() gets called before the render() method and componentDidMount() is called after the render() method.

Overall the mounting process has 4 lifecycle methods. They are invoked in the following order:

  • constructor()
  • componentWillMount()
  • render()
  • componentDidMount()

But what about the update lifecycle of a component that happens when the state or the props change? Overall it has 5 lifecycle methods in the following order:

  • componentWillReceiveProps()
  • shouldComponentUpdate()
  • componentWillUpdate()
  • render()

Por último, pero no menos importante es el ciclo de vida de desmontaje. Tiene sólo un método de ciclo de vida: componentWillUnmount().

Last but not least there is the unmounting lifecycle. It has only one lifecycle method: componentWillUnmount().

Después de todo, usted no necesita saber todos estos métodos de ciclo de vida desde el principio. podes ser intimidante sin embargo, no va a utilizar todos ellos - incluso en un maduro React aplicación. Sin embargo, es bueno saber que cada método de ciclo de vida se podes utilizar para los casos de uso específicos:

After all, you don’t need to know all of these lifecycle methods from the beginning. It can be intimidating yet you will not use all of them - even in a mature React application. Still, it is good to know that each lifecycle method can be used for specific use cases:

  • constructor (apoyos) - Se llama cuando el componente se inicializa. Se podes establecer un estado de componente inicial y se unen métodos clase útil durante ese método ciclo de vida.
  • constructor(props) - It is called when the component gets initialized. You can set an initial component state and bind useful class methods during that lifecycle method.
  • componentWillMount () - Se llama antes de que el render() método de ciclo de vida. Es por eso que se podría utilizar para establecer el estado del componente interno, porque no va a desencadenar una segunda versión del componente. Por lo general, se recomienda usar el constructor() de establecer el estado inicial.
  • componentWillMount() - It is called before the render() lifecycle method. That’s why it could be used to set internal component state, because it will not trigger a second rendering of the component. Generally it is recommend to use the constructor() to set the initial state.
  • render () - El método de ciclo de vida es obligatoria y devuelve los elementos como una salida del componente. El método debes ser puro y por lo tanto no debes modificar el estado del componente. Se pone una entrada como accesorios y el estado y devuelve un elemento.
  • render() - The lifecycle method is mandatory and returns the elements as an output of the component. The method should be pure and therefore shouldn’t modify the component state. It gets an input as props and state and returns an element.
  • componentDidMount () - Se llama sólo una vez cuando el componente montado. Ese es el momento perfecto para hacer una solicitud asincrónica para obtener los datos de un API. Los datos captados se almacenan en obtener el estado de los componentes internos para mostrarla en el render() método de ciclo de vida.
  • componentDidMount() - It is called only once when the component mounted. That’s the perfect time to do an asynchronous request to fetch data from an API. The fetched data would get stored in the internal component state to display it in the render() lifecycle method.
  • componentWillReceiveProps (nextProps) - El método se llama ciclo de vida durante un ciclo de vida de actualización. Como entrada a obtener los siguientes accesorios. podes diff los siguientes apoyos con los apoyos anteriores ( this.props) para aplicar un comportamiento diferente en función de la diferenciación. Además, podes definir el estado en base a los siguientes accesorios.
  • componentWillReceiveProps(nextProps) - The lifecycle method is called during an update lifecycle. As input you get the next props. You can diff the next props with the previous props (this.props) to apply a different behavior based on the diff. Additionally you can set state based on the next props.
  • shouldComponentUpdate (nextProps, del estado siguiente) - Siempre se llama cuando las actualizaciones de los componentes debido a otro o de los apoyos de los cambios. Lo va a usar en aplicaciones madurReact para optimizaciones de rendimiento. En función de un valor lógico que regrese de este método de ciclo de vida, el componente y todos tus hijos se hacen o no se hacen en un ciclo de vida de actualización. podes evitar que el método de ciclo de vida de procesamiento de un componente.
  • shouldComponentUpdate(nextProps, nextState) - It is always called when the component updates due to state or props changes. You will use it in mature React applications for performance optimizations. Depending on a boolean that you return from this lifecycle method, the component and all its children will render or will not render on an update lifecycle. You can prevent the render lifecycle method of a component.
  • componentWillUpdate (nextProps, del estado siguiente) - El método de ciclo de vida se invoca inmediatamente antes del render() método. Ya tiene los siguientes accesorios y siguiente estado a su disposición. Podes utilizar el método como última oportunidad para realizar los preparativos antes de que el método render es ejecutado. Tenga en cuenta que no se podes desencadenar setState() más. Si desea calcular el estado en base a los siguientes apoyos, tenes que utilizar componentWillReceiveProps().
  • componentWillUpdate(nextProps, nextState) - The lifecycle method is immediately invoked before the render() method. You already have the next props and next state at your disposal. You can use the method as last opportunity to perform preparations before the render method gets executed. Note that you cannot trigger setState() anymore. If you want to compute state based on the next props, you have to use componentWillReceiveProps().
  • componentDidUpdate (prevProps, prevState) - El método de ciclo de vida se invoca inmediatamente después de la render() método. Se podes utilizar como una oportunidad para llevar a cabo operaciones DOM o para realizar otras peticiones asincrónicas.
  • componentDidUpdate(prevProps, prevState) - The lifecycle method is immediately invoked after the render() method. You can use it as opportunity to perform DOM operations or to perform further asynchronous requests.
  • componentWillUnmount () - Se llama antes de destruir su componente. Se podes utilizar el método de ciclo de vida para llevar a cabo las tareas de limpieza.
  • componentWillUnmount() - It is called before you destroy your component. You can use the lifecycle method to perform any clean up tasks.

Las constructor() y render() los métodos de ciclo de vida que ya son utilizados por usted. Estos son los métodos de ciclo de vida de uso común para los componentes de clase ES6. En realidad, el render() se requiere método, de lo contrario no devolver una instancia de componente.

The constructor() and render() lifecycle methods are already used by you. These are the commonly used lifecycle methods for ES6 class components. Actually the render() method is required, otherwise you wouldn’t return a component instance.
