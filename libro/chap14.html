ES6 const and let

Creo que se ha notado que declaramos la variable helloWorld con var. JavaScript ES6 viene con dos opciones más para declarar tus variables: const e let. En JavaScript ES6 que rara vez se encuentra var más. Vamos a obtener una explicación de const y let:

I guess you noticed that we declared the variable helloWorld with var. JavaScript ES6 comes with two more options to declare your variables: const and let. In JavaScript ES6 you will rarely find var anymore. Let’s get some explanation for const and let:

Una variable declarada con const no podes ser reasignado o re-declarado. No podes ser mutado (cambiado, modificado). Abrazas estructuras de datos inmutables mediante el uso de la misma. Una vez definida la estructura de datos, no se podes cambiar.

A variable declared with const cannot be re-assigned or re-declared. It cannot get mutated (changed, modified). You embrace immutable data structures by using it. Once the data structure is defined, you cannot change it.

Una variable declarada con let podes conseguir mutado.

A variable declared with let can get mutated.

Se podría usar cuando se tendría que volver a asignar una variable.

You would use it when you would need to re-assign a variable.

Sin embargo, hay que tener cuidado con const. Una variable declarada con const no podes quedar modificado. Pero cuando la variable es una array o un objeto, el valor que posee podes quedar alterado. El valor que posee no es inmutable.

However, you have to be careful with const. A variable declared with const cannot get modified. But when the variable is an array or object, the value it holds can get altered. The value it holds is not immutable.

Pero cuando usar cada declaración? Hay diferentes opiniones sobre el uso. Es mejor utilizar const siempre que pueda. Indica que desea mantener la estructura de datos inmutables a pesar de que los valores de los objetos y las matrices pueden quedar modificado. Si desea modificar la variable, se podes utilizar let.

But when to use each declaration? There are different opinions about the usage. I suggest to use const whenever you can. It indicates that you want to keep your data structure immutable even though values in objects and arrays can get modified. If you want to modify your variable, you can use let.

Inmutabilidad es abrazado en React y su ecosistema. Es por eso que const debe ser su opción por defecto cuando se define una variable. Aun así, en objetos complejos los valores dentro pueden quedar modificado. Tenga cuidado acerca de este comportamiento.

Immutability is embraced in React and its ecosystem. That’s why const should be your default choice when you define a variable. Still, in complex objects the values within can get modified. Be careful about this behavior.

En tu aplicaciónse debes utilizar const más var.

In your application you should use const over var.
