Si estás buscando más extensiones para tu aplicación, puedo recomendar varias rutas de aprendizaje:

If you are looking for further extensions for your application, I can recommend several learning paths:

  • Organización Código: En el camino de la lectura del libro que se encontró un capítulo sobre la organización del código. Se podría aplicar estos cambios ahora, si no lo ha hecho todavía. Se organizará tus componentes en los archivos y carpetas (módulos) estructurados. Además ayuda a entender y aprender los principios de división de código, reutilización, mantenimiento y diseño de la API del módulo.
  • Code Organization: On your way reading the book you came across one chapter about Code Organization. You could apply these changes now, if you haven’t done it yet. It will organize your components in structured files and folders (modules). In addition it helps to understand and learn the principles of code splitting, reusability, maintainability and module API design.
  • Las solicitudes asíncronas: Se podes sustituir la API fetch nativa con las alternativas de terceros para realizar peticiones asíncronas: superagente o axios. No hay una solución perfecta para hacer peticiones asíncronas. Sin embargo, mediante el intercambio de los edificios cuadras a la redondReact , a tomar la experiencia de lo poderoso que podes ser tener esta flexibilidad. En los frameworks que se adhieren a una solución, en un ecosistema flexible como el React se pueden intercambiar las soluciones.
  • Asynchronous Requests: You can substitute the native fetch API with third party alternatives to perform asynchronous requests: superagent or axios. There is no perfect solution to make asynchronous requests. But by exchanging the buildings blocks around React, you make the experience how powerful it can be to have this flexibility. In frameworks you stick to one solution, in a flexible ecosystem like React you can exchange the solutions.
  • La verificación de tipos: En un capítulo que ha usado React PropTypes para definir interfaces de componentes. Es una buena práctica para evitar que los insectos. Pero los PropTypes sólo se verifican en tiempo de ejecución. Podes ir un paso más allá para introducir la comprobación de tipos estáticos en tu editor. Letra de imprenta es un método popular. Pero en el ecosistemReact personas prefieren utilizar Flow. Puedo recomendar para dar flujo de un tiro.
  • Type Checking: In one chapter you have used React PropTypes to define component interfaces. It is a good practice to prevent bugs. But the PropTypes are only checked on runtime. You can go one step further to introduce static type checking in your editor. TypeScript is one popular approach. But in the React ecosystem people prefer to use Flow. I can recommend to give Flow a shot.
  • Herramientas con webpack y Babel: Vos ha configurado tu aplicacióncon crear una aplicación React. En algún momento, cuando aprendió React, que desea aprender las herramientas a su alrededor para configurar su propio proyecto sin crear una aplicación React. Puedo recomendar seguir una configuración mínima con webpack y Babel. Después se podía configuración ESLint a seguir un estilo de código unificado en tu aplicación.
  • Tooling with Webpack and Babel: You have setup your application with create-react-app. At some point, when you learned React, you want to learn the tooling around it to setup your own project without create-react-app. I can recommend to follow a minimal setup with Webpack and Babel. Afterward you could setup ESLint to follow a unified code style in your application.
  • Gestión del Estado: Ha utilizado React setState a administrar el estado de los componentes internos. Eso es un comienzo perfecto. Sin embargo, en una aplicación de escala va a experimentar los límites del estado de los componentes internos. Por lo tanto tenes bibliotecas de administración de estado de terceros como Redux o MOBX.
  • State Management: You have used React setState to manage internal component state. That’s a perfect start. However, in a scaling application you will experience the limits of internal component state. Therefore you have third party state management libraries like Redux or MobX.
  • Enrutamiento: Se podes implementar enrutamiento para tu aplicacióncon React-enrutador. Hasta el momento sólo tiene una página en la aplicación. React Router te ayuda a tener varias páginas en diferentes direcciones URL.
  • Routing: You can implement routing for your application with react-router. So far you only have one page in your application. React Router helps you to have multiple pages across different URLs.
